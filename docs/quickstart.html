Quick Start Guide

Get up and running with DockSkope in under 5 minutes. This guide will walk you through the essential steps to start monitoring your Docker containers.

Prerequisites: Make sure you have Docker installed and running on your system.

Step 1: Install DockSkope

The fastest way to get started is using Docker Compose:

# Clone the repository
git clone https://github.com/DockSkope.git
cd DockSkope

# Start DockSkope
docker-compose -f docker-compose.production.yml up -d

This will start all DockSkope components automatically. Wait a few moments for all services to initialize.

Step 2: Access the Web Interface

Open your browser and navigate to:

http://localhost:3000

You should see the DockSkope login page. Use the default credentials:

  • Username: admin
  • Password: admin
Security Note: Change the default password immediately after your first login.

Step 3: Explore the Dashboard

After logging in, you'll see the main dashboard with:

  • System Overview: CPU, memory, and disk usage
  • Container List: All running containers with their status
  • Recent Alerts: Any triggered alert notifications
  • Quick Actions: Start, stop, and restart containers

Step 4: Monitor Your First Container

Let's create a test container to see DockSkope in action:

# Run a simple test container
docker run -d --name test-nginx nginx:latest

# Check the container in DockSkope
# Navigate to the Containers section in the web interface

You should now see your test container in the DockSkope interface with real-time metrics.

Step 5: Set Up Your First Alert

Create an alert to monitor your container's CPU usage:

  1. Go to the Alerts section in the sidebar
  2. Click Create New Alert
  3. Configure the alert:
    • Name: High CPU Usage
    • Metric: CPU Usage
    • Condition: Greater than
    • Threshold: 80%
    • Duration: 2 minutes
  4. Save the alert rule

Step 6: View Container Logs

Access real-time logs for your containers:

  1. Go to the Containers section
  2. Click on your test container
  3. Navigate to the Logs tab
  4. View real-time log output

Common Commands

Here are some useful commands for managing DockSkope:

Command Description
docker-compose -f docker-compose.production.yml up -d Start DockSkope in the background
docker-compose -f docker-compose.production.yml down Stop DockSkope
docker-compose -f docker-compose.production.yml logs View DockSkope logs
docker-compose -f docker-compose.production.yml restart Restart DockSkope services
Congratulations! You've successfully set up DockSkope and are now monitoring your Docker containers. The system will automatically collect metrics and provide insights into your containerized applications.
