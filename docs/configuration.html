Configuration Guide

Learn how to configure DockSkope for your specific environment and requirements. This guide covers all configuration options and settings.

Note: Most configuration changes require restarting the DockSkope services to take effect.

Environment Variables

DockSkope uses environment variables for configuration. You can set these in your .env file or directly in your Docker Compose configuration.

Core Configuration

Variable Description Default Required
DOCKER_HOST Docker daemon socket path unix:///var/run/docker.sock No
BACKEND_PORT Backend server port 8080 No
FRONTEND_PORT Frontend server port 3000 No
AGENT_PORT Agent communication port 9090 No
JWT_SECRET JWT token secret key your-secret-key Yes

Database Configuration

Variable Description Default Required
INFLUXDB_URL InfluxDB connection URL http://localhost:8086 No
INFLUXDB_TOKEN InfluxDB authentication token your-token No
INFLUXDB_ORG InfluxDB organization dockskope No
INFLUXDB_BUCKET InfluxDB bucket name dockskope_metrics No

Email Configuration

Variable Description Default Required
SMTP_HOST SMTP server hostname localhost No
SMTP_PORT SMTP server port 587 No
SMTP_USERNAME SMTP username No
SMTP_PASSWORD SMTP password No
SMTP_FROM_EMAIL From email address for notifications noreply@dockskope.com No

Configuration Files

In addition to environment variables, DockSkope uses configuration files for more complex settings.

Agent Configuration

The agent configuration file is located at agent/config.yaml: 

# Agent Configuration
agent:
  name: "default-agent"
  host: "localhost"
  port: 9090
  
# Metrics Collection
metrics:
  collection_interval: 30s
  retention_days: 30
  
# Docker Configuration
docker:
  socket_path: "/var/run/docker.sock"
  api_version: "1.41"
  
# Logging
logging:
  level: "info"
  format: "json"
  output: "stdout"

Backend Configuration

The backend configuration file is located at backend/config.yaml: 

# Server Configuration
server:
  host: "0.0.0.0"
  port: 8080
  read_timeout: 30s
  write_timeout: 30s
  
# Database Configuration
database:
  type: "influxdb"
  url: "http://localhost:8086"
  token: "${INFLUXDB_TOKEN}"
  org: "${INFLUXDB_ORG}"
  bucket: "${INFLUXDB_BUCKET}"
  
# Authentication
auth:
  jwt_secret: "${JWT_SECRET}"
  token_expiry: 24h
  
# Alerting
alerts:
  check_interval: 30s
  notification_timeout: 10s

Web Interface Configuration

Configure the web interface through the settings panel:

General Settings

  • Theme: Light/Dark mode
  • Language: Interface language
  • Time Zone: Display timezone
  • Refresh Interval: Auto-refresh frequency

Dashboard Configuration

  • Default Dashboard: Choose which dashboard to show on login
  • Widget Layout: Customize dashboard widget positions
  • Metrics Display: Configure which metrics to show by default

Security Configuration

Configure security settings for your DockSkope installation:

Authentication

  • Password Policy: Minimum password requirements
  • Session Timeout: Automatic logout after inactivity
  • Multi-factor Authentication: Enable 2FA for users

Network Security

  • HTTPS: Enable SSL/TLS encryption
  • Firewall Rules: Configure network access
  • IP Whitelisting: Restrict access to specific IPs

Performance Configuration

Optimize DockSkope performance for your environment:

Resource Limits

Component CPU Limit Memory Limit Storage
Backend 1 CPU 512MB 1GB
Frontend 0.5 CPU 256MB 100MB
Agent 0.5 CPU 256MB 50MB
InfluxDB 1 CPU 1GB 10GB

Metrics Collection

  • Collection Interval: How often to collect metrics (default: 30s)
  • Retention Policy: How long to keep historical data (default: 30 days)
  • Data Compression: Enable/disable data compression

Monitoring Configuration

Configure what and how to monitor:

Container Monitoring

  • Auto-discovery: Automatically monitor new containers
  • Excluded Containers: Containers to ignore
  • Custom Labels: Add custom labels for organization

Metrics to Collect

  • CPU Usage: Enable/disable CPU monitoring
  • Memory Usage: Enable/disable memory monitoring
  • Network I/O: Enable/disable network monitoring
  • Disk I/O: Enable/disable disk monitoring
  • Custom Metrics: Define custom metrics to collect

Alerting Configuration

Configure alerting rules and notifications:

Default Alert Rules

  • High CPU Usage: Alert when CPU > 80% for 5 minutes
  • High Memory Usage: Alert when memory > 90% for 5 minutes
  • Container Down: Alert when container stops unexpectedly
  • Disk Space: Alert when disk usage > 85%

Notification Channels

  • Email: Configure SMTP settings
  • Slack: Webhook URL for Slack notifications
  • Webhook: Custom HTTP endpoints
  • PagerDuty: PagerDuty integration

Backup and Recovery

Configure backup settings for your DockSkope data:

Automatic Backups

  • Backup Schedule: Daily/weekly/monthly backups
  • Backup Location: Local or remote storage
  • Retention: How long to keep backups
  • Encryption: Encrypt backup files

Manual Backup

# Create a backup
docker-compose -f docker-compose.production.yml exec backend ./backup.sh

# Restore from backup
docker-compose -f docker-compose.production.yml exec backend ./restore.sh backup_file.tar.gz

Configuration Examples

Here are some common configuration scenarios:

Development Environment

# .env file for development
DOCKER_HOST=unix:///var/run/docker.sock
BACKEND_PORT=8080
FRONTEND_PORT=3000
JWT_SECRET=dev-secret-key
INFLUXDB_URL=http://localhost:8086
SMTP_HOST=localhost
SMTP_PORT=1025

Production Environment

# .env file for production
DOCKER_HOST=unix:///var/run/docker.sock
BACKEND_PORT=8080
FRONTEND_PORT=3000
JWT_SECRET=your-super-secure-jwt-secret-key
INFLUXDB_URL=http://influxdb:8086
INFLUXDB_TOKEN=your-influxdb-token
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=your-email@gmail.com
SMTP_PASSWORD=your-app-password
SMTP_FROM_EMAIL=noreply@yourcompany.com

Multi-Host Setup

# Agent configuration for remote host
agent:
  name: "remote-host-1"
  host: "192.168.1.100"
  port: 9090
  
docker:
  socket_path: "/var/run/docker.sock"
  
metrics:
  collection_interval: 30s
  retention_days: 30

Configuration Validation

Validate your configuration before applying changes:

# Validate configuration
docker-compose -f docker-compose.production.yml config

# Test configuration
docker-compose -f docker-compose.production.yml exec backend ./validate-config.sh
Important: Always backup your configuration before making changes, especially in production environments.
