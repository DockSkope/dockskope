Alerting System

Learn how to set up and manage intelligent alerts in DockSkope to stay informed about your container environment.

Proactive Monitoring: DockSkope's alerting system helps you identify and respond to issues before they impact your applications.

Overview

DockSkope's alerting system provides comprehensive monitoring and notification capabilities:

  • Real-time Alerts: Instant notifications when conditions are met
  • Flexible Rules: Customizable alert conditions and thresholds
  • Multiple Channels: Email, webhooks, Slack, and custom integrations
  • Alert Management: Acknowledge, resolve, and track alert history
  • Escalation Policies: Automatic escalation for critical alerts
  • Alert Suppression: Prevent alert spam during maintenance

Alert Rules

Alert rules define when and how alerts are triggered:

Rule Components

  • Name: Descriptive name for the alert rule
  • Description: Detailed description of what the alert monitors
  • Metric: The metric to monitor (CPU, memory, etc.)
  • Condition: Comparison operator (>, <, >=, <=, ==, !=)
  • Threshold: The value that triggers the alert
  • Duration: How long the condition must be true
  • Containers: Which containers to monitor
  • Notifications: How to notify when alert triggers

Supported Metrics

Metric Description Unit Example Threshold
cpu_usage CPU utilization percentage % 80%
memory_usage Memory usage percentage % 90%
memory_usage_bytes Memory usage in bytes bytes 1GB
network_rx_bytes Network bytes received bytes 100MB
network_tx_bytes Network bytes transmitted bytes 100MB
disk_read_bytes Disk bytes read bytes 50MB
disk_write_bytes Disk bytes written bytes 50MB
container_status Container status string stopped

Creating Alert Rules

Follow these steps to create effective alert rules:

Step 1: Define the Alert

  1. Go to the Alerts section in the sidebar
  2. Click Create New Alert
  3. Enter a descriptive name and description

Step 2: Configure the Condition

  1. Select the metric to monitor
  2. Choose the comparison operator
  3. Set the threshold value
  4. Define the duration (how long the condition must be true)

Step 3: Select Containers

  • All Containers: Monitor all containers
  • Specific Containers: Select individual containers
  • Container Groups: Use container labels or groups

Step 4: Configure Notifications

  • Email: Send email notifications
  • Webhook: Send HTTP POST requests
  • Slack: Send Slack messages
  • Custom: Integrate with other systems

Alert Management

Effectively manage and track your alerts:

Alert States

  • Active: Alert is currently triggered
  • Acknowledged: Alert has been acknowledged but not resolved
  • Resolved: Alert condition has returned to normal
  • Suppressed: Alert is temporarily suppressed

Alert Actions

  • Acknowledge: Mark alert as acknowledged
  • Resolve: Manually resolve the alert
  • Suppress: Temporarily suppress the alert
  • Escalate: Escalate to higher priority

Alert History

  • Historical Data: View past alert occurrences
  • Trend Analysis: Analyze alert patterns
  • Performance Metrics: Track alert system performance
  • Audit Trail: Complete audit trail of alert actions

Escalation Policies

Set up automatic escalation for critical alerts:

Escalation Levels

  • Level 1: Initial notification to on-call engineer
  • Level 2: Escalate to senior engineer after timeout
  • Level 3: Escalate to manager for critical issues
  • Level 4: Emergency escalation procedures

Escalation Rules

  • Time-based: Escalate after specific time periods
  • Condition-based: Escalate based on alert severity
  • Pattern-based: Escalate for recurring alerts
  • Manual Escalation: Allow manual escalation

Alert Suppression

Prevent alert spam during maintenance or known issues:

Suppression Types

  • Time-based: Suppress during scheduled maintenance
  • Container-based: Suppress for specific containers
  • Alert-based: Suppress specific alert types
  • Manual Suppression: Temporary manual suppression

Suppression Rules

  • Maintenance Windows: Define maintenance periods
  • Known Issues: Suppress during known problems
  • Testing Periods: Suppress during testing
  • Recovery Periods: Suppress during recovery

Best Practices

Follow these best practices for effective alerting:

Alert Design

  • Meaningful Names: Use descriptive alert names
  • Clear Descriptions: Explain what the alert monitors
  • Appropriate Thresholds: Set realistic thresholds
  • Reasonable Durations: Avoid too short or long durations

Notification Strategy

  • Right Channel: Use appropriate notification channels
  • Right People: Notify the right people
  • Right Time: Consider time zones and schedules
  • Right Frequency: Avoid notification fatigue

Alert Maintenance

  • Regular Review: Review and update alert rules
  • Performance Monitoring: Monitor alert system performance
  • Documentation: Document alert configurations
  • Testing: Test alert rules regularly

API Integration

Manage alerts programmatically using the API:

Alert Management API

  • Create Alerts: POST /api/v1/alerts
  • List Alerts: GET /api/v1/alerts
  • Update Alerts: PUT /api/v1/alerts/{id}
  • Delete Alerts: DELETE /api/v1/alerts/{id}
  • Alert Events: GET /api/v1/alerts/events
