General Questions

What is DockSkope?

DockSkope is a lightweight, agent-based Docker container monitoring and alerting system. It provides real-time insights into your containerized applications with features like resource monitoring, alerting, log management, and container lifecycle management.

How is DockSkope different from other monitoring tools?

DockSkope is specifically designed for Docker containers with a focus on simplicity and efficiency. Unlike complex enterprise monitoring solutions, DockSkope provides:

  • Lightweight agent architecture
  • Zero-configuration setup
  • Real-time container metrics
  • Built-in alerting system
  • Modern web interface
  • RESTful API for integration

What operating systems does DockSkope support?

DockSkope supports Linux, macOS, and Windows (with WSL2). The agent and backend are written in Go, making them cross-platform compatible.

Is DockSkope free to use?

Yes, DockSkope is open-source and free to use. You can find the source code on GitHub.

Installation & Setup

What are the system requirements?

Minimum requirements:

  • Docker version 20.10 or higher
  • 2GB RAM (4GB recommended)
  • 1GB free disk space
  • Internet connection for initial setup

How do I install DockSkope?

The easiest way is using Docker Compose:

git clone https://github.com/DockSkope.git
cd DockSkope
docker-compose -f docker-compose.production.yml up -d

For detailed instructions, see our Installation Guide.

Can I install DockSkope without Docker?

Yes, you can install each component separately. See the Installation Guide for manual installation instructions.

How do I access the web interface?

After installation, the web interface is available at http://localhost:3000 by default.

Monitoring & Metrics

What metrics does DockSkope collect?

DockSkope collects the following metrics:

  • CPU Usage: Percentage of CPU utilization
  • Memory Usage: Memory consumption in MB/GB
  • Network I/O: Bytes sent and received
  • Disk I/O: Read and write operations
  • Container Status: Running, stopped, paused, etc.
  • System Metrics: Host system resource usage

How often are metrics collected?

By default, metrics are collected every 30 seconds. You can configure this interval in the agent configuration.

How long is historical data retained?

Historical data is retained for 30 days by default. You can configure retention policies in the InfluxDB configuration.

Can I monitor containers on multiple hosts?

Yes, DockSkope supports multi-host monitoring. You can deploy agents on multiple Docker hosts and they will all report to the central backend.

Alerting

What types of alerts can I create?

You can create alerts based on:

  • CPU usage thresholds
  • Memory usage thresholds
  • Container status changes
  • Network I/O thresholds
  • Custom metrics

How do I receive alert notifications?

DockSkope supports multiple notification channels:

  • Email notifications
  • Webhook integrations (Slack, Discord, etc.)
  • Custom HTTP endpoints

Can I set up different alert thresholds for different containers?

Yes, you can create container-specific alert rules or apply rules to groups of containers.

Security

Is DockSkope secure?

DockSkope includes several security features:

  • JWT-based authentication
  • Role-based access control
  • HTTPS support
  • Secure API endpoints
  • Environment variable configuration

How do I secure the installation?

For production deployments:

  • Change default passwords
  • Use HTTPS
  • Configure firewall rules
  • Use strong JWT secrets
  • Regular security updates

Does DockSkope require root access?

No, DockSkope can run with limited permissions. The agent only needs access to the Docker socket and the ability to read system metrics.

Troubleshooting

Why can't I see my containers?

Common causes:

  • Docker daemon not running
  • Agent not connected to backend
  • Permission issues with Docker socket
  • Network connectivity problems

Check the Troubleshooting Guide for detailed solutions.

How do I check if DockSkope is running properly?

You can check the status of each component:

# Check backend
curl http://localhost:8080/health

# Check agent
curl http://localhost:9090/health

# Check frontend
curl http://localhost:3000

Where are the logs stored?

Logs are stored in the following locations:

  • Backend: backend/logs/
  • Agent: agent/logs/
  • Docker: docker-compose logs

How do I update DockSkope?

To update DockSkope:

# Pull latest changes
git pull origin main

# Rebuild and restart
docker-compose -f docker-compose.production.yml down
docker-compose -f docker-compose.production.yml up -d --build

API & Integration

Is there an API available?

Yes, DockSkope provides a comprehensive RESTful API. See the API Reference for complete documentation.

Can I integrate DockSkope with other tools?

Yes, DockSkope can integrate with:

  • CI/CD pipelines
  • Monitoring dashboards (Grafana, etc.)
  • Notification services (Slack, Discord, etc.)
